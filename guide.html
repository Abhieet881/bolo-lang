सीखने की गाइड

शुरुआत से आगे तक

नौ पाठ, आसान से कठिन तक। हर पाठ ख़ुद टाइप करके चलाओ — देखकर नहीं, हाथ से लिखकर सीखो।

पाठ १ · शुरुआती

चर और गणित

क्यों सीखें: गणना करने और जानकारी को याद रखने के लिए हमें चर (Variables) की आवश्यकता होती है। यह एक डिब्बे की तरह होते हैं जहाँ हम कोई भी मान सुरक्षित रख सकते हैं।

मान से एक चर बनाओ, उसमें संख्या रखो, और छापो से परिणाम दिखाओ।

मान क = 10
मान ख = 5
छापो (क + ख)
इसे बदल कर देखो

चर ख का मान बदलकर १५ (15) करें और फिर गुणा (*) ऑपरेटर का उपयोग करके परिणाम छापें। क्या आपका परिणाम 150 आया?

पाठ २ · शुरुआती

शर्त (if / else)

क्यों सीखें: परिस्थितियों के हिसाब से अलग फैसला लेने के लिए हमें 'शर्तों' (Conditionals) की आवश्यकता होती है। इससे हमारा प्रोग्राम समझदार और उपयोगी बनता है।

अगर किसी शर्त को जाँचता है; अगर वह सच नहीं है, तो नहीं_तो वाला हिस्सा चलता है।

मान संख्या = 7
अगर (संख्या % 2 == 0) {
    छापो ("सम संख्या")
} नहीं_तो {
    छापो ("विषम संख्या")
}
इसे बदल कर देखो

संख्या का मान बदलकर १२ (12) करें। क्या आउटपुट "सम संख्या" में बदल गया? फिर कोई विषम संख्या लिखकर देखें।

पाठ ३ · मध्यम

लूप (जब_तक)

क्यों सीखें: जब किसी कार्य को बार-बार दोहराना हो, तो 'लूप' (Loops) का उपयोग किया जाता है। इससे हमारे समय और मेहनत की बहुत बचत होती है।

जब_तक एक शर्त के सच होने तक बार-बार चलता है। यहाँ 1 से 10 तक जोड़ा जा रहा है।

मान योग = 0
मान i = 1
जब_तक (i <= 10) {
    योग = योग + i
    i = i + 1
}
छापो (योग)
इसे बदल कर देखो

शर्त को i <= 20 में बदलें और योग करने की सीमा को 1 से 20 तक बढ़ाएं। चलाने से पहले अनुमान लगाएं कि नया योग क्या होगा!

पाठ ४ · मध्यम

नेस्टेड शर्तें (FizzBuzz)

क्यों सीखें: जटिल परिस्थितियों में निर्णय लेने के लिए हमें एक शर्त के अंदर दूसरी शर्त लगानी पड़ती है। नेस्टेड शर्तें (Nested Conditions) हमें ऐसा जटिल लॉजिक हल करने की आज़ादी देती हैं।

एक लूप के अंदर कई अगर/नहीं_तो होने पर भी कोड साफ़ रहता है।

मान i = 1
जब_तक (i <= 15) {
    अगर (i % 15 == 0) {
        छापो ("FizzBuzz")
    } नहीं_तो अगर (i % 3 == 0) {
        छापो ("Fizz")
    } नहीं_तो अगर (i % 5 == 0) {
        छापो ("Buzz")
    } नहीं_तो {
        छापो (i)
    }
    i = i + 1
}
इसे बदल कर देखो

लूप की सीमा को 15 से बढ़ाकर 30 करें। देखें कि 30 पर "FizzBuzz" प्रिंट होता है या नहीं और 23 जैसी संख्या पर क्या प्रिंट होता है।

पाठ ५ · उन्नत

फ़ंक्शन और recursion

क्यों सीखें: बार-बार इस्तेमाल होने वाले कोड को हम 'फ़ंक्शन' का नाम दे देते हैं। फ़ंक्शन द्वारा खुद को दोबारा बुलाना 'Recursion' कहलाता है, जो जटिल समस्याओं को आसान बनाता है।

कार्य से एक फ़ंक्शन बनता है, जो वापस से मान लौटाता है — और ख़ुद को भी बुला सकता है।

कार्य फ़ैक्टोरियल(n) {
    अगर (n <= 1) {
        वापस 1
    }
    वापस n * फ़ैक्टोरियल(n - 1)
}
छापो (फ़ैक्टोरियल(5))
इसे बदल कर देखो

फ़ैक्टोरियल(5) की जगह फ़ैक्टोरियल(6) को बुलाकर देखें। क्या उत्तर 720 आया?

पाठ ६ · उन्नत

array के साथ काम करना

क्यों सीखें: कई मानों की सूची को एक साथ सहेजने के लिए 'Array' का उपयोग किया जाता है। इससे डेटा की पूरी सूची को एक ही स्थान पर संभालना आसान हो जाता है।

लंबाई और के_लिए को साथ में इस्तेमाल करके किसी सूची के हर तत्व पर काम करो।

मान फल = ["आम", "केला", "सेब"]
मान i = 0
जब_तक (i < लंबाई(फल)) {
    छापो (फल[i])
    i = i + 1
}
इसे बदल कर देखो

जोड़ो (append) फ़ंक्शन का उपयोग करके सूची में एक नया फल (जैसे "संतरा") जोड़ें और देखें कि क्या वह भी प्रिंट होता है।

पाठ ७ · मध्यम

string जोड़ना (concatenation)

क्यों सीखें: प्रोग्रामिंग में अक्षरों और शब्दों (strings) को आपस में जोड़ने और उनकी लंबाई मापने की बहुत ज़रूरत होती है। बोलो में प्लस (+) ऑपरेटर से दो स्ट्रिंग्स को जोड़ा जा सकता है, और लंबाई फ़ंक्शन से अक्षरों की संख्या पता की जा सकती है।

दो शब्दों या वाक्यों को एक साथ जोड़ने के लिए प्लस (+) ऑपरेटर का उपयोग करें।

मान नाम = "अमित"
मान बधाई = "नमस्ते, " + नाम + "!"
छापो (बधाई)
छापो (लंबाई(नाम))
इसे बदल कर देखो

बधाई के अंत में कोई और शब्द जोड़कर देखें या अपना खुद का नाम लिखकर लंबाई जाँचें।

पाठ ८ · मध्यम

नेस्टेड लूप (Nested Loops)

क्यों सीखें: जब हमें किसी ग्रिड (जैसे शतरंज का बोर्ड, 2D ग्रिड) या किसी पैटर्न पर काम करना होता है, तो हमें लूप के अंदर दूसरा लूप चलाना पड़ता है। इसे नेस्टेड लूप कहते हैं।

के_लिए लूप के अंदर एक और के_लिए लूप चलाकर स्टार (*) का एक सुंदर त्रिभुज (triangle) पैटर्न छापें।

के_लिए (मान i = 1; i <= 5; i = i + 1) {
    मान पंक्ति = ""
    के_लिए (मान j = 1; j <= i; j = j + 1) {
        पंक्ति = पंक्ति + "*"
    }
    छापो (पंक्ति)
}
इसे बदल कर देखो

बाहरी लूप की सीमा को ५ से बढ़ाकर ८ करें और आउटपुट का पैटर्न देखें। क्या त्रिभुज बड़ा हो गया?

पाठ ९ · उन्नत

एक छोटा प्रोजेक्ट: ग्रेड कैलकुलेटर

क्यों सीखें: अब तक सीखी हुई सभी चीज़ों (चर, शर्त, फ़ंक्शन) को मिलाकर एक छोटा प्रोजेक्ट बनाते हैं। यह फ़ंक्शन छात्र के अंकों के आधार पर उसका ग्रेड तय करता है।

सभी बुनियादी अवधारणाओं का एक साथ उपयोग करके एक व्यावहारिक ग्रेडिंग सिस्टम का निर्माण करें।

कार्य ग्रेड_कैलकुलेटर(अंक) {
    अगर (अंक >= ९०) {
        वापस "A"
    } नहीं_तो अगर (अंक >= ८०) {
        वापस "B"
    } नहीं_तो अगर (अंक >= ७०) {
        वापस "C"
    } नहीं_तो {
        वापस "F"
    }
}

मान छात्र_के_अंक = ८५
मान ग्रेड = ग्रेड_कैलकुलेटर(छात्र_के_अंक)
छापो ("छात्र का ग्रेड है: " + ग्रेड)
इसे बदल कर देखो

छात्र_के_अंक को बदलकर ६५ करके देखें और परिणाम का अनुमान लगाएं। फिर F ग्रेड की जगह D ग्रेड के लिए एक नई शर्त जोड़ें।

सावधानी

आम गलतियाँ (Common Mistakes)

बोलो में कोड लिखते समय शुरुआत में ये गलतियाँ होना आम हैं। इनसे बचें:

गलती १: अगर/जब_तक की शर्त में कोष्ठक (Parentheses) भूलना

बोलो में अगर या जब_तक की शर्त को हमेशा कोष्ठक () के अंदर लिखना आवश्यक है।

# गलत:
अगर संख्या > ५ { ... }

# सही:
अगर (संख्या > ५) { ... }
गलती २: तुलना के लिए = का उपयोग करना

एकल बराबर = का उपयोग किसी चर में मान डालने (assign) के लिए होता है। दो मानों की तुलना करने के लिए हमेशा दोहरे बराबर == का उपयोग करें।

# गलत:
अगर (संख्या = ५) { ... }

# सही:
अगर (संख्या == ५) { ... }
गलती ३: घुमावदार कोष्ठक (Curly Braces) छोड़ना

शर्तों, लूप और फ़ंक्शन के ब्लॉक को शुरू और समाप्त करने के लिए हमेशा घुमावदार कोष्ठक {} लगाना ज़रूरी है, भले ही उसमें सिर्फ़ एक ही लाइन हो।

# गलत:
अगर (संख्या > ५) छापो("बड़ी है")

# सही:
अगर (संख्या > ५) {
    छापो("बड़ी है")
}
गलती ४: मान (let) के बिना चर घोषित करना

जब भी आप पहली बार कोई नया चर बनाते हैं, तो उसके नाम के पहले मान (let) लिखना अनिवार्य है।

# गलत:
संख्या = १०

# सही:
मान संख्या = १०
गलती ५: फ़ंक्शन के बाद सेमीकोलन लगाना

फ़ंक्शन की परिभाषा के घुमावदार कोष्ठक } के बाद सेमीकोलन लगाने की आवश्यकता नहीं होती है।

# गलत:
कार्य जोड़ो(क, ख) { वापस क + ख; };

# सही:
कार्य जोड़ो(क, ख) { वापस क + ख }
आगे क्या

अभ्यास के लिए विचार

हर पाठ पूरा करने के बाद, ख़ुद से यह लिखकर देखो:

#अभ्यास
1किसी संख्या को उल्टा करना (reverse a number)
2array में सबसे बड़ी संख्या ढूँढना
3दो संख्याओं का GCD निकालना (recursion से)
4एक सरल कैलकुलेटर फ़ंक्शन बनाना

💡 यदि आप इन अभ्यासों के काम करने वाले कोड और हल किए गए समाधान देखना चाहते हैं, तो हमारे उदाहरण (Examples) पेज पर जाएँ।
